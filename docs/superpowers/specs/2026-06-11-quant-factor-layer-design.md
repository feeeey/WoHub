# 量化因子挖掘与分析层 — 分阶段设计

日期：2026-06-11
状态：方向草案（实施前需用户确认优先级；本文档定义里程碑边界，不附带实施计划）

## 定位

WoHub 路线图的第三阶段：在「安全加固 → 测试网验证 → 自动化层」之后，向
**量化因子挖掘与量化分析**演进。约束（来自用户交易方法）：

- **纯技术分析**：因子只来自价格、成交量、衍生指标。不引入新闻/情绪/链上数据源。
- **以损定仓**：任何自动化策略的仓位均由止损距离反推，复用 `position_plan.py`。

## 已有的地基（不重复建设）

| 已有资产 | 量化层如何复用 |
|----------|----------------|
| `klines/`（Candle、14 种形态、4 级分类、分形枢轴、ATR） | 因子计算的原料与第一批因子 |
| `signals` + `outcomes` 表（信号 1h/4h/24h 前向收益追踪） | 现成的事件-收益样本库，IC/胜率分析直接可算 |
| `snapshots` 表（信号时刻价格/量/资金费率） | 事件时点的横截面特征 |
| Pine 筛选器任务管道 | 信号事件的持续生产者（每天累积样本） |
| `trading/`（bracket + SL 恢复 + 风险仓位计算） | 自动交易执行端，量化层只产出 direction/confidence |

## 分阶段里程碑

### P1 — 信号效果分析（先收割已有数据）

目的：回答「现有筛选器信号到底有没有用」。零新数据源。

- `backend/quant/outcome_stats.py`（纯函数）：按 indicator × timeframe × signal_type
  分组，计算样本数、1h/4h/24h 胜率、平均/中位收益、收益分布分位数。
- API `GET /quant/signal-stats`（参数：时间窗、最小样本数）。
- 前端：信号统计页（表格 + 简单分布图），标注样本不足的格子。
- 验收：能用历史 signals/outcomes 数据排出「最有效的筛选器×周期组合」。

### P2 — 因子库与 IC 分析

目的：把"信号"泛化为"因子值"，建立因子 → 前向收益的相关性度量。

- `backend/quant/factors.py`（纯函数，输入 `list[Candle]`，输出 float 或分类值）。
  第一批因子（全部可由现有模块导出）：
  - 动量：ROC(n)、收盘价相对 n 期高低点位置
  - 波动：ATR 比率（短/长）、K 线实体/影线比
  - 结构：距最近分形枢轴的 ATR 归一化距离、枢轴年龄
  - 形态/分类：14 形态 one-hot、4 级分类状态
  - 量能：成交量 z-score、量价背离
- `backend/quant/ic.py`：对历史 K 线滚动计算因子值与 n 期前向收益的
  Spearman IC、IC 均值/标准差/IR、分桶收益单调性。
- 数据获取：直接拉 klines（fetcher 已支持 1500 根），无需新表；计算结果缓存到
  新表 `factor_runs`（参数哈希 → 结果 JSON）。
- 验收：任选 symbol×interval，能输出因子 IC 排名表。

### P3 — 回测骨架（事件驱动，bar-close 撮合）

目的：把"因子有相关性"变成"策略有期望收益"，含成本模型。

- `backend/quant/backtest.py`：单 symbol、bar-close 成交、taker 费率 + 滑点参数、
  以损定仓仓位（复用 `position_plan.compute_plan` 的纯数学部分）、
  强制止损/止盈/超时退出三选一。
- 输出：权益曲线、年化收益、Sharpe、最大回撤、胜率、盈亏比、暴露时间。
- **过拟合防线（不可省略）**：参数网格搜索必须走 walk-forward
  （训练窗选参 → 紧邻验证窗评估），报告只看验证窗拼接结果；
  样本 < 100 笔的组合直接标记不可信。
- 验收：对 P2 排名靠前的因子组合跑出含成本的 walk-forward 报告。

### P4 — 接入自动交易（与自动化层汇合）

目的：量化层产出标准化信号，自动化层消费。

- 信号 schema：`{symbol, interval, direction: long|short, confidence: 0-1, factors: {...}}`。
- `auto_trade` 任务类型（路线图已规划）：默认 dry-run，只记录"本应执行的交易"；
  实盘开关受全局 kill-switch + 日亏熔断 + 测试网先行约束。
- 验收：dry-run 模式下自动信号与人工 Trade 页下单走同一条
  plan → bracket（含 SL 恢复）管道。

## 显式不做

- 不做多交易所横截面因子（先 Binance USDT-M 单所）。
- 不做 tick 级回测/高频（bar-close 已匹配现有信号粒度）。
- 不引入重量级依赖（vectorbt/zipline 等）——样本量级用 numpy/纯 Python 足够，
  且与现有"无 ccxt、手写客户端"的工程口味一致。
- 不做机器学习因子组合（先把线性 IC 与分桶单调性做扎实）。

## 顺序依赖

P1 仅依赖已有数据，可立即开始。P2 不依赖 P1 但共享 quant/ 模块骨架。
P3 依赖 P2 的因子库。P4 依赖自动化层 phase 1（kill-switch + 熔断）先落地。
